\hyperlink{classMatthiasMullie_1_1Minify_1_1JS}{JS} can have /-\/delimited regular expressions, like\+: /ab+c/.match(string).

The content inside the regex can contain characters that may be confused for \hyperlink{classMatthiasMullie_1_1Minify_1_1JS}{JS} code\+: e.\+g. it could contain whitespace it needs to match \& we don\textquotesingle{}t want to strip whitespace in there.

The regex can be pretty simple\+: we don\textquotesingle{}t have to care about comments, (which also use slashes) because \hyperlink{classMatthiasMullie_1_1Minify_1_1JS_ac3943c7955686b4663841127d34904aa}{strip\+Comments()} will have stripped those already.

This method will replace all string content with simple R\+E\+G\+EX\# placeholder text, so we\textquotesingle{}ve rid all regular expressions from characters that may be misinterpreted. Original regex content will be saved in \$this-\/$>$extracted and after doing all other minifying, we can restore the original content via restore\+Regex() \index{Matthias\+Mullie\+::\+Minify\+::\+JS@{Matthias\+Mullie\+::\+Minify\+::\+JS}!get\+Keywords\+For\+Regex@{get\+Keywords\+For\+Regex}}

Strip whitespace.

We won\textquotesingle{}t strip {\itshape all} whitespace, but as much as possible. The thing that we\textquotesingle{}ll preserve are newlines we\textquotesingle{}re unsure about. Java\+Script doesn\textquotesingle{}t require statements to be terminated with a semicolon. It will automatically fix missing semicolons with A\+SI (automatic semi-\/ colon insertion) at the end of line causing errors (without semicolon.)

Because it\textquotesingle{}s sometimes hard to tell if a newline is part of a statement that should be terminated or not, we\textquotesingle{}ll just leave some of them alone.
